# Brainstorming de Design - Portal SMMB

## Contexto
O projeto é um portal corporativo para gestão de contratos, cronogramas e documentação técnica do desenvolvimento de um CRM para a Administradora Mutual. O público-alvo são executivos, gestores de projeto e equipe jurídica. O tom deve ser profissional, seguro, tecnológico e organizado.

<idea>
  **Design Movement**: Neo-Brutalism Corporativo (Suavizado)
  
  **Core Principles**:
  1. **Clareza Radical**: Informação apresentada sem distrações, com alto contraste e hierarquia óbvia.
  2. **Estrutura Modular**: Todo o conteúdo é organizado em "blocos" ou "cards" claramente definidos, lembrando documentos físicos organizados.
  3. **Autenticidade**: Uso de tipografia monoespaçada para dados técnicos e serifada para textos legais, transmitindo seriedade.
  4. **Funcionalidade Primeiro**: Navegação direta, sem menus ocultos ou animações excessivas que atrasem o acesso.

  **Color Philosophy**:
  - **Paleta**: Preto (#1A1A1A), Branco (#FFFFFF), Cinza Concreto (#F0F0F0) e Azul Institucional (#0056B3) como acento principal (alinhado com Potere/Mutual).
  - **Razão**: Transmite solidez, transparência e foco no conteúdo. O azul traz a confiança corporativa necessária.

  **Layout Paradigm**:
  - **Bento Grid**: Layout baseado em grade modular (bento box), onde cada seção (contratos, cronogramas, anexos) ocupa um espaço retangular definido.
  - **Sidebar Persistente**: Navegação lateral esquerda fixa para acesso rápido a todas as seções, similar a um IDE ou ferramenta de gestão profissional.

  **Signature Elements**:
  - **Bordas Definidas**: Uso de bordas finas (1px) para separar todos os elementos, criando uma sensação de organização rigorosa.
  - **Ícones Técnicos**: Ícones de linha fina (Lucide) que remetem a arquivos, código e processos.
  - **Badges de Status**: Etiquetas coloridas (verde, amarelo, cinza) para indicar status de documentos (Assinado, Pendente, Versão Final).

  **Interaction Philosophy**:
  - **Feedback Tátil**: Hover states claros com mudança de cor de fundo ou borda.
  - **Transições Rápidas**: Animações curtas (150ms) e secas, sem "bounciness".
  - **Pré-visualização**: Modais ou painéis laterais para preview rápido de documentos sem sair da página principal.

  **Animation**:
  - **Fade-in Sutil**: Elementos aparecem suavemente ao carregar.
  - **Highlight**: Foco visual no elemento ativo da navegação.

  **Typography System**:
  - **Títulos**: *Inter* (Bold/Semibold) para clareza e modernidade.
  - **Corpo**: *Inter* (Regular) para leitura confortável.
  - **Dados/Código**: *JetBrains Mono* ou *Roboto Mono* para números, datas e referências técnicas.
</idea>
</text>

  - **Frosted Glass**: Efeito de vidro fosco em headers e modais.

## Decisão Final
**Design Escolhido**: **Neo-Brutalism Corporativo (Suavizado)**

**Justificativa**: Este estilo combina perfeitamente com a necessidade de um portal de **gestão de contratos e documentação técnica**. A estrutura modular (Bento Grid) é ideal para organizar diferentes tipos de conteúdo (PDFs, cronogramas, diagramas) de forma igualitária e acessível. A clareza radical e a autenticidade transmitem a segurança jurídica e técnica que o projeto exige. O uso do Azul Institucional alinha-se com a identidade da Administradora Mutual/Potere.

**Adaptações para o Projeto**:
- Manteremos a paleta sóbria mas usaremos o azul da marca para CTAs e destaques.
- A navegação lateral será fundamental para alternar entre Contratos, Anexos e Cronogramas.
- Os "cards" do Bento Grid servirão como pontos de entrada para visualizar os documentos detalhados.
